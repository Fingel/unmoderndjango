Inline Validation

Sometimes the user benefits from immediate form validation feedback. Especially in longer forms, it can be annoying to fill them out only to learn that some value to entered at the top won't work. Client-side validation can handle a lot of these cases but some validation can only be performed server-side.

Do you need inline validation? In many cases, inline server-side validation is unnecessary. The client-side form validation API is powerful and can handle almost all cases that don't rely on data from the server. And even if you do need server-side validation, sometimes a full form submit isn't a huge deal - especially with simple forms that are submitted asynchronously.
